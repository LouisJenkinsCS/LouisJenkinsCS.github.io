4 /*
5  * This Linked List implementation is meant for general and generic usage, and is
6  * by no means a replacement for more specific-purpose data structures. On top
7  * of that, this Linked List should be thread-safe, however iteration is not thread-ideal.
8  *
9  * The Linked List allows you to add items in a sorted order and even sort the entire list
10  * by means of a passed comparator, and whether or not it chooses to add items in a sorted
11  * order depends on whether or not the callback is NULL or not. Hence, if you want
12  * to add an item to the list without having to deal with creating and passing a comparator
13  * function, you can easily do like so...
14  *
15  * Linked_List_add(list, item, NULL);
16  *
17  * Inversely, you may add an item in sorted order by doing so like this:
18  *
19  * Linked_List_add(list, item, comparator);
20  *
21  * Upon deletion, you have the option to delete the item as well as the node by passing
22  * a comparator, similar to above.
23  *
24  * Linked_List_remove_at(list, 0, NULL);
25  *
26  * or
27  *
28  * Linked_List_remove_at(list, 0, free);
29  *
30  * Iteration is thread-safe, however not thread-ideal. As there is only one iterator,
31  * if multiple threads attempt to maneuver it, you may not like the results. Adding,
32  * removing and sorting is thread safe and thread ideal, as multiple threads can safely,
33  * but in a synchronized way. However, retrieval is is thread-safe and allows multiple
34  * threads to read over items in the list, as the Linked List uses a rwlock.
35  *
36  * This Linked List, once again is general purpose, and should not be used if performance is of utmost
37  * importance. The sort is slow for elements over 2,500, however if you keep it under
38  * 1,000, it definitely will do it's job.
39  */
