4 /*
5  * To summarize, a thread pool is a pool of threads that initialized at creation,
6  * and perform tasks; by initializing a pool a threads beforehand, it greatly reduces
7  * the overhead of creating and deleting a thread for each task.
8  *
9  * Now that that's clear, how does my thread pool differ? My thread pool features
10  * selective pausing of tasks and pausing all tasks and resuming them at a later time
11  * either by a timer or manually resuming the thread pool. My thread pool also features
12  * the ability to obtain the result from a task and is optional.
13  * It also gives the ability to wait for all tasks to finish, and alternatively to
14  * wait up to a certain amount of time.
15  *
16  * The thread pool currently is a static member, hence there can be only one at
17  * any given time, however at a later date there will instead be a static table
18  * of thread pools to allow multiple, however as that would complicate development,
19  * I decided to push that off to a later date.
20  *
21  * The thread pool also is static in another sense. The amount of threads will always
22  * have the same amount of threads at runtime, and will only be reduced (to 0) on deletion.
23  *
24  * The Thread Pool can be initialized with the below macros, used for bitmasking.
25  * For example, to add a task where you want no result, you would do the following:
26  *
27  * Thread_Pool_Add_Task(function, arguments, TP_NO_RESULT);
28  *
29  * If you wanted to add a task where you did want the result, but you wanted
30  * it of high priority and not being able to pause midway, you would do the following:
31  *
32  * Thread_Pool_Add_Task(function, arguments, TP_NO_PAUSE | TP_HIGH_PRIORITY);
33  *
34  * Alternatively, if you want to add a normal task (medium priority), you do the following:
35  *
36  * Thread_Pool_Add_Task(function, arguments, TP_NONE);
37  *
38  * Note that you do not need to specify everything, as in whether you want the result
39  * or not, as there are set defaults, listed below. These defaults are there to make
40  * adding general tasks easier.
41  *
42  * Defaults:
43  *
44  * - Result returned
45  * - Will pause mid-task.
46  * - Medium Priority task.
47  */
